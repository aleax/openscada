1.1 Policy (started from 0.8)

OpenSCADA versions form into the two branches:

  • Work: development-work version with the night builds for some number of Linux distributions and directly specified by the subscription for support.
1+rNNNN, where "NNNN" — the SVN (SubVersion) revision number.
  • LTS (Long Term Support): formal and public release of the last "Work" version with support up to the next iteration: "Work" -> "LTS" (zero update number), "Work"++.
0.9.U.S, where:
"U" — regular updating number, 2-4 month. After the regular updating period, all fixes and improvements from the "Work" version will be merged to "LTS", and also, those that break some API compatibility, but do not break the functionality and data compatibility; and U++.
"S" (doesn't implemented yet) — service updating number. A service update (S++) version will be built, automatically or manually (for specific), for any hot fixes which allowed only for consumers subscribed to the official support.
